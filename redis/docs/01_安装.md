# Redis 安装指南

本文介绍两种在 Linux 上安装 Redis 的方式：

1. 在 Ubuntu/Debian 上通过 Redis 官方 APT 仓库安装。
2. 使用 Docker 官方镜像运行 Redis。

官方参考文档：

- [在 Linux 上安装 Redis](https://redis.io/docs/latest/operate/oss_and_stack/install/install-redis/install-redis-on-linux/)
- [Redis Docker 官方镜像](https://hub.docker.com/_/redis)

## 方式一：在 Ubuntu/Debian 上安装

### 1. 安装仓库依赖

```bash
sudo apt-get update
sudo apt-get install -y lsb-release curl gpg
```

### 2. 添加 Redis 官方 APT 仓库

导入仓库签名密钥：

```bash
curl -fsSL https://packages.redis.io/gpg \
  | sudo gpg --dearmor -o /usr/share/keyrings/redis-archive-keyring.gpg
sudo chmod 644 /usr/share/keyrings/redis-archive-keyring.gpg
```

添加与当前系统版本对应的软件源：

```bash
echo "deb [signed-by=/usr/share/keyrings/redis-archive-keyring.gpg] https://packages.redis.io/deb $(lsb_release -cs) main" \
  | sudo tee /etc/apt/sources.list.d/redis.list
```

### 3. 安装并启动 Redis

```bash
sudo apt-get update
sudo apt-get install -y redis
sudo systemctl enable --now redis-server
```

Redis 的主要配置文件通常位于 `/etc/redis/redis.conf`。修改配置后需要重启服务：

```bash
sudo systemctl restart redis-server
```

### 4. 验证安装

```bash
redis-cli ping
```

正常情况下会返回：

```text
PONG
```

## 方式二：使用 Docker 安装

本节假设 Linux 上已经安装并启动 Docker Engine，并且以下命令默认使用具有 `sudo` 权限的用户执行。

### 1. 检查 Docker

```bash
docker --version
```

如果当前用户没有访问 Docker 的权限，可在下列 Docker 命令前添加 `sudo`。

### 2. 拉取镜像

```bash
docker pull redis:latest
```

### 3. 启动 Redis 容器

```bash
docker run -d \
  --name redis \
  --restart unless-stopped \
  -p 127.0.0.1:6379:6379 \
  -v redis-data:/data \
  redis:latest \
  redis-server --appendonly yes
```

参数说明：

- `--name redis`：将容器命名为 `redis`。
- `--restart unless-stopped`：Docker 重启后自动恢复容器，手动停止的情况除外。
- `-p 127.0.0.1:6379:6379`：只允许宿主机本地访问 Redis。
- `-v redis-data:/data`：将 Redis 数据保存到 Docker 具名卷中。
- `--appendonly yes`：启用 AOF 持久化。

### 4. 验证容器

在容器中执行 Redis 命令：

```bash
docker exec redis redis-cli ping
```

正常情况下会返回 `PONG`。

## 安全提示

- 不要将未配置认证的 Redis `6379` 端口直接暴露到公网。
- 本文的 Docker 示例仅将端口绑定到 `127.0.0.1`。如需远程访问，应同时配置防火墙、Redis ACL 或密码认证，并限制允许访问的客户端。
- 生产环境应使用固定版本、定期备份数据，并在升级前完成兼容性测试。
- 不要在命令行或版本库中保存明文密码。
