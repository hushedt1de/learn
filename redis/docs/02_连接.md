## Redis 连接

使用 `redis-cli` 连接 Redis 的基本格式如下：

```bash
redis-cli [OPTIONS] [COMMAND [ARG ...]]
```

连接本机默认 Redis 实例：

```bash
redis-cli
```

也可以在连接时直接执行命令：

```bash
redis-cli PING
```

常用连接参数及默认值：

| 参数 | 作用 | 默认值 |
| --- | --- | --- |
| `-h <host>` | Redis 服务器地址 | `127.0.0.1` |
| `-p <port>` | Redis 服务器端口 | `6379` |
| `--user <username>` | ACL 用户名，需要与密码一起使用 | 未设置（仅使用 `-a` 时认证 Redis 的 `default` 用户） |
| `-a <password>` | 认证密码 | 未设置 |
| `-n <db>` | 连接后选择的数据库编号 | `0` |
| `--tls` | 使用 TLS 连接 | 关闭 |
| `--cacert <file>` | 用于验证服务器证书的 CA 证书文件 | 未设置 |
| `-u <uri>` | 使用 Redis URI 指定连接信息 | 未设置 |

指定主机、端口和数据库：

```bash
redis-cli -h 192.0.2.10 -p 6379 -n 0
```

如果 Redis 启用了认证，建议通过 `REDISCLI_AUTH` 环境变量传入密码，避免密码出现在命令历史和进程列表中：

```bash
REDISCLI_AUTH='your-password' redis-cli \
  -h 192.0.2.10 -p 6379 --user app
```

也可以使用 URI 连接：

```bash
redis-cli -u 'redis://app:your-password@192.0.2.10:6379/0'
```

URI 格式为 `redis://[[username]:password@]host[:port][/db]`。TLS 连接使用 `rediss://` 方案，或在普通参数中添加 `--tls`。将密码直接写入 URI 也可能泄露密码，仅建议在受控环境中使用。

如果 Redis 服务在 Docker 容器中，可在宿主机上通过映射端口连接：

```bash
redis-cli -p 6379
```

如果宿主机没有安装 `redis-cli`，可直接在容器内连接：

```bash
docker exec -it redis redis-cli
```
