# MySQL 数据操作语言

数据操作语言（Data Manipulation Language，DML）用于新增、修改和删除表中的数据。

本章示例主要使用以下学生表：

```sql
CREATE TABLE students (
  id BIGINT UNSIGNED PRIMARY KEY AUTO_INCREMENT,
  student_no VARCHAR(20) NOT NULL UNIQUE,
  name VARCHAR(50) NOT NULL,
  gender ENUM('男', '女', '未知') NOT NULL DEFAULT '未知',
  age TINYINT UNSIGNED,
  score DECIMAL(5, 2),
  enrolled_on DATE NOT NULL,
  updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP
    ON UPDATE CURRENT_TIMESTAMP,
  CHECK (age IS NULL OR age <= 150),
  CHECK (score IS NULL OR score BETWEEN 0 AND 100)
);
```

## 1. 新增数据

### 1.1 新增单行数据

```sql
INSERT INTO 表名 (字段1, 字段2, ...)
VALUES (值1, 值2, ...);
```

示例：

```sql
INSERT INTO students (student_no, name, gender, age, score, enrolled_on)
VALUES ('20260001', '张三', '男', 18, 92.50, '2026-09-01');
```

字段列表与值列表必须一一对应。未出现在字段列表中的字段：

- 定义了默认值时，使用默认值。
- 允许为 `NULL` 时，使用 `NULL`。
- 使用 `AUTO_INCREMENT` 时，由 MySQL 生成自增值。
- 定义为 `NOT NULL` 且没有默认值时，在严格 SQL 模式下会插入失败。

可以省略字段列表，表示按表中的字段顺序为所有字段提供值。但更建议明确写出字段名。

### 1.2 批量新增数据

在一条语句中写入多行数据，通常比逐行执行多条 `INSERT` 效率更高：

```sql
INSERT INTO students
  (student_no, name, gender, age, score, enrolled_on)
VALUES
  ('20260003', '王五', '男', 19, 85.50, '2026-09-01'),
  ('20260004', '赵六', '女', 18, 95.00, '2026-09-01'),
  ('20260005', '钱七', '未知', NULL, NULL, '2026-09-02');
```

单次批量写入的数据量还会受 SQL 长度、网络开销和 `max_allowed_packet` 等因素影响。大量数据应合理分批，不宜把所有数据组装成一条过大的 SQL 语句。

### 1.3 将查询结果写入表（待审批）

`INSERT ... SELECT` 可以把查询结果写入另一张已存在的表：

```sql
INSERT INTO excellent_students (student_id, student_no, name, score)
SELECT id, student_no, name, score
FROM students
WHERE score >= 90;
```

目标字段的数量、顺序和数据类型应与查询结果匹配。这条语句只写入数据，不会自动复制原表的主键、索引和约束。

### 1.4 处理唯一键冲突（待审批）

插入的主键或唯一键已存在时，普通 `INSERT` 会执行失败。

如果希望出现唯一键冲突时改为更新已有记录，可以使用 `ON DUPLICATE KEY UPDATE`：

```sql
INSERT INTO students (student_no, name, gender, enrolled_on)
VALUES ('20260001', '张三', '男', '2026-09-01')
ON DUPLICATE KEY UPDATE
  name = '张三',
  gender = '男';
```

该语句可能因主键或任意唯一键冲突而进入更新分支，因此表中存在多个唯一约束时应特别注意。

`INSERT IGNORE` 会忽略某些可恢复错误并继续执行：

```sql
INSERT IGNORE INTO students (student_no, name, enrolled_on)
VALUES ('20260001', '重复学号', '2026-09-01');
```

它不仅会处理重复键，还可能将其他错误转换为警告，容易隐藏数据质量问题。使用后应通过以下语句检查警告：

```sql
SHOW WARNINGS;
```

## 2. 修改数据

### 2.1 基本语法

```sql
UPDATE 表名
SET 字段1 = 值1,
    字段2 = 值2,
    ...
[WHERE 条件];
```

修改指定学号的学生信息：

```sql
UPDATE students
SET age = 19,
    score = 93.50
WHERE student_no = '20260001';
```

`WHERE` 用于限定需要修改的行。省略 `WHERE` 时，表中所有行都会被修改：

```sql
UPDATE students
SET score = 0;
```

对全表执行修改应当是明确的业务需求，执行前应仔细确认。

### 2.2 根据原值计算新值

`SET` 后面可以使用字段原值进行计算。例如，将所有不低于 60 分的成绩增加 2 分，并保证结果不超过 100：

```sql
UPDATE students
SET score = LEAST(score + 2, 100)
WHERE score >= 60;
```

`NULL` 参与普通算术运算后，结果仍然是 `NULL`。需要把 `NULL` 当作指定值时，可以使用 `COALESCE`：

```sql
UPDATE students
SET score = COALESCE(score, 0) + 5
WHERE student_no = '20260005';
```

### 2.3 将字段恢复为默认值或空值

```sql
UPDATE students
SET gender = DEFAULT,
    age = NULL
WHERE student_no = '20260001';
```

`DEFAULT` 表示使用字段定义中的默认值。将字段设为 `NULL` 前，应确认该字段没有 `NOT NULL` 约束。

### 2.4 关联修改（待审批）

MySQL 可以根据其他表的数据修改目标表。例如，根据成绩表更新学生的成绩：

```sql
UPDATE students AS s
JOIN exam_results AS e ON e.student_id = s.id
SET s.score = e.score
WHERE e.exam_name = '2026 学年期末考试';
```

关联条件缺失或不正确时，可能会匹配过多行。执行关联修改前，应先使用相同的 `JOIN` 和 `WHERE` 条件进行查询，确认匹配结果。

### 2.5 排序并限制修改行数（待审批）

MySQL 允许对单表 `UPDATE` 使用 `ORDER BY` 和 `LIMIT`：

```sql
UPDATE students
SET score = 60
WHERE score < 60
ORDER BY score DESC, id
LIMIT 10;
```

- `ORDER BY` 决定先处理哪些行。
- `LIMIT` 限制本次最多匹配的行数。

分批修改时应使用稳定且可确定的排序，通常需要把主键加入 `ORDER BY`。多表 `UPDATE` 不支持使用 `ORDER BY` 和 `LIMIT` 限制更新顺序和行数。

## 3. 删除数据

### 3.1 基本语法

```sql
DELETE FROM 表名
[WHERE 条件];
```

删除指定学号的学生：

```sql
DELETE FROM students
WHERE student_no = '20260006';
```

`WHERE` 用于限定需要删除的行。省略 `WHERE` 时，表中所有行都会被删除：

```sql
DELETE FROM students;
```

`DELETE` 删除全表数据后仍然保留表结构，并且通常不会重置 `AUTO_INCREMENT` 计数器。

### 3.2 排序并限制删除行数（待审批）

MySQL 允许对单表 `DELETE` 使用 `ORDER BY` 和 `LIMIT`。例如，删除 100 条最早的未激活记录：

```sql
DELETE FROM pending_students
WHERE status = 'inactive'
ORDER BY created_at, id
LIMIT 100;
```

这种写法适合分批清理大量历史数据，可以缩短单个事务的执行时间。

### 3.3 关联删除（待审批）

可以根据其他表的数据删除目标表中的记录。例如，删除 2020 年以前入学的学生的选课记录：

```sql
DELETE sc
FROM student_courses AS sc
JOIN students AS s ON s.id = sc.student_id
WHERE s.enrolled_on < '2020-01-01';
```

`DELETE sc` 指定只删除别名为 `sc` 的表中的数据，`students` 表只用于匹配条件。

多表删除可以同时指定多个目标表，但风险较高，应优先考虑外键约束的 `ON DELETE CASCADE` 或在事务中按明确顺序删除。

### 3.4 外键对删除的影响（待审批）

删除父表数据时，子表外键上定义的引用操作会决定 MySQL 如何处理相关数据：

- `ON DELETE RESTRICT` 或 `NO ACTION`：存在子表引用时拒绝删除父表记录。
- `ON DELETE CASCADE`：删除父表记录时，自动删除相关子表记录。
- `ON DELETE SET NULL`：删除父表记录时，将子表外键设为 `NULL`。

使用级联删除前，应确认关联层级和影响范围，避免一次删除意外影响大量数据。

## 4. 安全地修改和删除数据（待审批）

### 4.1 先查询，再操作

执行 `UPDATE` 或 `DELETE` 前，建议先使用相同的 `WHERE` 条件查询将要受影响的数据：

```sql
SELECT id, student_no, name, score
FROM students
WHERE score IS NULL;

UPDATE students
SET score = 0
WHERE score IS NULL;
```

检查时应重点确认：

- 条件是否会匹配过多或过少的数据。
- `AND` 和 `OR` 组合时的优先级是否正确，必要时使用括号。
- 判断空值时是否使用了 `IS NULL` 或 `IS NOT NULL`，而不是 `= NULL` 或 `<> NULL`。
- 日期时间范围的边界是否正确。

### 4.2 使用事务保护操作

对 InnoDB 表执行重要修改或删除时，可以使用事务先检查结果：

```sql
START TRANSACTION;

UPDATE students
SET score = score + 1
WHERE score BETWEEN 60 AND 99;

SELECT ROW_COUNT() AS affected_rows;

COMMIT;
```

如果发现操作结果不符合预期，应在执行 `COMMIT` 之前回滚：

```sql
ROLLBACK;
```

`COMMIT` 会提交事务，之后不能再通过 `ROLLBACK` 撤销。事务会持有锁和占用资源，检查后应及时提交或回滚。

`ROW_COUNT()` 返回上一条 DML 语句影响的行数，应紧跟在目标语句之后调用。客户端连接选项可能影响 `UPDATE` 返回“实际改变的行数”还是“匹配的行数”。

### 4.3 安全更新模式

某些 MySQL 客户端会启用 `SQL_SAFE_UPDATES` 模式。在该模式下，既没有使用键字段限定条件，也没有使用 `LIMIT` 的 `UPDATE` 和 `DELETE` 可能被拒绝，常见错误为 1175。

查看当前会话设置：

```sql
SELECT @@SESSION.sql_safe_updates;
```

应优先完善筛选条件，而不是直接关闭安全模式。确实需要在当前会话关闭时，可以执行：

```sql
SET SESSION SQL_SAFE_UPDATES = 0;
```

设置只对当前连接生效。操作完成后可以重新启用：

```sql
SET SESSION SQL_SAFE_UPDATES = 1;
```

## 5. DML 语句对比

| 语句 | 主要作用 | `WHERE` | 主要风险 |
| ---- | -------- | ------- | -------- |
| `INSERT` | 新增数据 | 不使用 | 重复写入、字段与值错位、触发唯一键冲突 |
| `UPDATE` | 修改已有数据 | 可选 | 条件错误或缺失时大量数据被误改 |
| `DELETE` | 删除已有数据 | 可选 | 条件错误或缺失时大量数据被误删 |

对重要数据执行 DML 语句时，应遵循“先确认影响范围、再在事务中执行、检查结果后提交”的原则，并定期备份关键数据。
