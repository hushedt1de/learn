# Docker 安装

本文记录 Docker Engine 的安装步骤，主要面向 Ubuntu 学习环境，其他 Linux 发行版请看 Docker 官方 Linux 安装入口。

官方参考文档：

- Docker Engine Linux 安装入口：https://docs.docker.com/engine/install/
- Docker Engine Ubuntu 安装文档：https://docs.docker.com/engine/install/ubuntu/
- Docker Engine Linux 安装后配置：https://docs.docker.com/engine/install/linux-postinstall/

## 1. 确认系统版本

```bash
cat /etc/os-release
```

如果输出中能看到 `Ubuntu`，则可以继续下面的步骤。

## 2. 卸载可能冲突的旧包

Linux 发行版的软件源里可能有 `docker.io`、`docker-compose` 等非 Docker 官方包。为了避免和 Docker 官方包冲突，先执行：

```bash
for pkg in docker.io docker-doc docker-compose docker-compose-v2 podman-docker containerd runc; do
  sudo apt remove -y "$pkg"
done
```

提示某些包没有安装是正常情况。

这一步不会删除已有的镜像、容器、数据卷和网络。如果机器之前装过 Docker，并且想彻底清空旧数据，需要另外处理 `/var/lib/docker/` 和 `/var/lib/containerd/`。

## 3. 安装基础依赖

```bash
sudo apt update
sudo apt install -y ca-certificates curl
```

创建 APT keyrings 目录：

```bash
sudo install -m 0755 -d /etc/apt/keyrings
```

## 4. 添加 Docker 官方 GPG key

```bash
sudo curl -fsSL https://download.docker.com/linux/ubuntu/gpg -o /etc/apt/keyrings/docker.asc
sudo chmod a+r /etc/apt/keyrings/docker.asc
```

## 5. 添加 Docker 官方 APT 源

```bash
sudo tee /etc/apt/sources.list.d/docker.sources <<EOF
Types: deb
URIs: https://download.docker.com/linux/ubuntu
Suites: $(. /etc/os-release && echo "${UBUNTU_CODENAME:-$VERSION_CODENAME}")
Components: stable
Architectures: $(dpkg --print-architecture)
Signed-By: /etc/apt/keyrings/docker.asc
EOF
```

更新软件源：

```bash
sudo apt update
```

## 6. 安装 Docker Engine

```bash
sudo apt install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
```

这些包分别提供 Docker Engine、Docker CLI、containerd、Buildx 插件和 Docker Compose v2 插件。安装后使用的是 `docker compose` 子命令，不是旧版独立命令 `docker-compose`。

安装完成后检查 Docker 服务：

```bash
sudo systemctl status docker
```

如果 Docker 没有运行，可以手动启动：

```bash
sudo systemctl start docker
```

可以显式设置 Docker 和 containerd 开机自启：

```bash
sudo systemctl enable docker.service
sudo systemctl enable containerd.service
```

## 7. 验证 Docker 可用

```bash
sudo docker --version
sudo docker compose version
sudo docker run hello-world
```

如果 `hello-world` 能正常输出，说明 Docker 基础环境可用。

## 8. 可选：配置 Docker Hub 镜像加速器

国内网络访问 Docker Hub 可能超时。如果出现类似错误：

```text
failed to resolve reference "docker.io/library/hello-world:latest"
dial tcp ...:443: i/o timeout
```

可以给 Docker 配置 registry mirror。编辑 Docker daemon 配置文件：

```bash
sudo mkdir -p /etc/docker
sudo vim /etc/docker/daemon.json
```

写入以下内容，把示例地址替换成你当前可用的镜像加速器地址：

```json
{
  "registry-mirrors": [
    "https://你的镜像加速器地址"
  ]
}
```

保存后重启 Docker：

```bash
sudo systemctl daemon-reload
sudo systemctl restart docker
```

检查配置是否生效：

```bash
sudo docker info
```

在输出中找到 `Registry Mirrors`，如果能看到你配置的地址，说明 Docker 已读取配置。

## 9. 可选：允许当前用户不加 sudo 使用 Docker

默认情况下，普通用户直接运行 `docker` 可能会遇到权限错误。如果这是个人学习机器，可以把当前用户加入 `docker` 用户组。

有些系统安装 Docker Engine 时会自动创建 `docker` 用户组。为了兼容没有自动创建的系统，可以先执行：

```bash
sudo groupadd docker
```

如果提示用户组已经存在，是正常情况。

把当前用户加入 `docker` 用户组：

```bash
sudo usermod -aG docker "$USER"
```

然后退出当前终端并重新登录，或重启系统。也可以在当前终端临时刷新用户组：

```bash
newgrp docker
```

重新登录后验证：

```bash
docker run hello-world
```

注意：加入 `docker` 用户组的用户基本等同于拥有较高的系统权限。只建议在个人学习环境中这样做；如果是多人服务器，应继续使用 `sudo docker ...` 或咨询管理员。

如果之前用 `sudo docker` 生成过当前用户目录下的 Docker 配置，改成免 `sudo` 后可能遇到 `~/.docker/config.json` 权限错误。可以修复属主和权限：

```bash
sudo chown "$USER":"$USER" "$HOME/.docker" -R
sudo chmod g+rwx "$HOME/.docker" -R
```
