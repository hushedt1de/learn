# Docker 基础概念

## 1. Docker 是什么

Docker 可以把应用和它需要的运行环境打包成镜像，再用镜像启动容器。容器和宿主机（运行 Docker 的机器）共享操作系统内核，但有独立的文件系统、进程空间和网络配置。

这样做的好处：

- 环境更容易复现。
- 多个软件可以用独立容器隔离运行。
- 删除容器后可以快速重建。

几个基础概念：

- 端口映射：把容器内端口暴露到宿主机端口。
- 数据卷：把容器数据保存到宿主机，避免容器删除后数据丢失。

## 2. 镜像

镜像是容器的模板，可以理解成一个只读的软件包。镜像里通常包含：

- 应用程序本身。
- 应用运行需要的依赖。
- 默认配置。
- 启动命令。
- 基础文件系统。

常见镜像示例：

```text
nginx:latest
mysql:8.4
redis:7
ubuntu:24.04
```

镜像名称通常由两部分组成：

```text
镜像名:标签
```

例如 `mysql:8.4` 中：

- `mysql` 是镜像名。
- `8.4` 是标签，通常表示镜像的版本。

如果不写标签，则默认使用 `latest`，它代表最新版本的镜像。生产环境下更建议固定一个具体版本。

## 3. 镜像仓库

仓库用于保存和分发镜像。

常见仓库：

- Docker Hub（Docker 官方维护的公共仓库）：https://hub.docker.com/
- 云厂商镜像仓库。
- 公司内部私有镜像仓库。

## 4. 容器

容器是镜像运行起来后的实例，可以启动、停止、删除。

镜像和容器的关系类似：

```text
镜像 -> 启动 -> 容器
```

同一个镜像可以启动多个容器：

```bash
docker run --name web-1 -d nginx:latest
docker run --name web-2 -d nginx:latest
```

容器运行在宿主机上，但默认和宿主机隔离：

- 容器内的文件不等于宿主机文件。
- 容器内的端口不等于宿主机端口。
- 容器内的进程默认只能在容器内看到。

这是因为容器有自己的：

- 进程空间。
- 文件系统。
- 网络配置。
- 环境变量。
- 启动命令。

因此每个容器都是独立的运行实例。

## 5. 端口映射

由于容器有自己的网络空间，外部无法直接访问到容器内的服务，因此需要借助 Docker 将容器端口暴露出来。

例如，Nginx 容器内部监听的 `80` 端口默认只在容器内可用，宿主机不能直接通过自己的 `80` 端口来访问它。此时可以把宿主机 `80` 端口映射到容器 `80` 端口，访问宿主机 `80` 端口的请求会被转发到容器内的 `80` 端口。

## 6. 数据卷和目录挂载

删除容器后，其内部文件通常也会随之消失。需要保留数据时，应使用数据卷或目录挂载。

数据卷是由 Docker 管理的虚拟目录，是 `容器内目录` 与 `宿主机目录` 之间映射的桥梁。可以方便操作容器内文件，或者持久化容器产生的数据。

创建数据卷时，Docker 会在宿主机的 `/var/lib/docker/volumes` 下创建一个真实目录，并让数据卷映射到该目录。

目录挂载指的是在容器内部挂载宿主机目录。

## 6. 环境变量

很多镜像会通过环境变量来完成初始化配置。

示例：

```bash
docker run --name mysql-demo \
  -e MYSQL_ROOT_PASSWORD=123456 \
  -d mysql:8.4
```

`-e MYSQL_ROOT_PASSWORD=123456` 会把环境变量传入容器。MySQL 官方镜像第一次初始化时会读取这个变量来设置 root 用户密码。

不同镜像支持的环境变量不同。使用具体镜像前，应查看该镜像的官方说明。

## 7. 容器网络

Docker 默认会创建一个 `bridge` 网络。普通容器如果不指定网络，通常会加入这个默认网络。

查看网络：

```bash
docker network ls
```

创建自定义网络：

```bash
docker network create app-net
```

在同一个自定义网络里，容器之间可以用容器名互相访问：

```bash
docker run --name redis-demo --network app-net -d redis:7
docker run --name app-demo --network app-net -d your-app:latest
```

如果 `app-demo` 需要访问 Redis，可以使用主机名 `redis-demo`。

学习单个容器时，主要掌握端口映射即可。学习多个容器协作时，再重点理解 Docker 网络。

## 8. Dockerfile

Dockerfile 是构建镜像的说明文件。

一个简单示例：

```dockerfile
FROM nginx:latest
COPY ./html /usr/share/nginx/html
```

含义：

- `FROM nginx:latest`：基于 Nginx 镜像构建新镜像。
- `COPY ./html /usr/share/nginx/html`：把当前目录下的 `html` 目录复制到镜像中。

构建镜像：

```bash
docker build -t my-nginx:1.0 .
```

运行镜像：

```bash
docker run --name my-nginx -p 8080:80 -d my-nginx:1.0
```

如果只是学习使用 MySQL、Redis、Nginx，通常先不需要自己写 Dockerfile。等需要把自己的应用打包成镜像时，再学习 Dockerfile。

## 9. Docker Compose

Docker Compose 用一个配置文件管理多个容器。

例如一个应用可能需要：

- Web 服务。
- MySQL。
- Redis。

如果全部用 `docker run`，命令会越来越长。Compose 可以把容器、端口、环境变量、数据卷、网络写到 `compose.yml` 中。

常用命令：

```bash
# 启动
docker compose up -d

# 查看服务
docker compose ps

# 查看日志
docker compose logs

# 停止并删除容器
docker compose down
```

Docker Compose v2 使用的是 `docker compose`，中间是空格。旧版 `docker-compose` 是单独命令，新项目优先使用 `docker compose`。
