# Docker 基本使用

## 1. 镜像命令

### 1.1 拉取镜像

```bash
docker pull mysql:latest
```

从镜像仓库中拉取 `mysql:latest` 这个镜像。

### 1.2 查看本机镜像

```bash
docker images
```

### 1.3 删除镜像

```bash
docker rmi mysql:latest
```

如果有容器正在使用这个镜像，需要先停止并删除对应容器。

## 2. 容器命令

### 2.1 运行容器

```bash
docker run -d \
    --name mysql \
    -p 3306:3306 \
    -e TZ=Asia/Shanghai \
    -e MYSQL_ROOT_PASSWORD=123 \
    mysql:latest
```

使用镜像创建并运行一个新容器。如果本机没有该镜像，通常会自动尝试从仓库中拉取。

参数说明：

- `-d`：后台运行容器。默认是前台运行，此时终端会被该容器占用，可以按 `Ctrl+C` 来停止容器。
- `--name mysql`：把容器命名为 `mysql`，名称不能重复。默认是随机名称。
- `-p 3306:3306`：设置端口映射。
- `-e KEY=VALUE`：设置环境变量。
- `mysql:latest`：使用的镜像。

容器创建后，就可以使用容器名称或容器 ID 来标识它。

### 2.2 停止容器

```bash
docker stop mysql
```

`docker stop` 会请求容器正常停止，停止后容器依旧存在。

### 2.3 启动已停止的容器

```bash
docker start mysql
```

`docker start` 只能启动已经存在的容器，不会创建新容器。它和 `docker run` 的区别是：

- `docker run`：创建一个新容器并启动。
- `docker start`：启动一个已经存在但停止了的容器。

### 2.4 重启容器

```bash
docker restart mysql
```

`docker restart` 等价于先停止再启动，常用于修改配置后重启服务。

### 2.5 删除容器

删除容器前，通常需要先将其停止：

```bash
docker stop mysql
docker rm mysql
```

也可以通过 `-f` 参数来强制删除容器：

```bash
docker rm -f mysql
```

删除容器并不会自动删除镜像。以后仍然可以用同一个镜像重新创建容器。

### 2.6 查看容器

```bash
docker ps -a
```

常见字段：

- `CONTAINER ID`：容器 ID。
- `IMAGE`：容器使用的镜像。
- `COMMAND`：容器启动命令。
- `STATUS`：容器状态。
- `PORTS`：端口映射。
- `NAMES`：容器名称。

参数说明：

- `-a`：all，显示所有容器，包括已经停止的容器。默认只显示正在运行的容器。

### 2.7 查看容器日志

```bash
docker logs -f mysql
```

参数说明：

- `-f`：follow，持续输出新日志。

### 2.8 进入容器执行命令

有时需要进入容器内部查看文件或执行命令，可以使用 `docker exec`。

```bash
docker exec -it mysql bash
```

参数说明：

- `docker exec`：在正在运行的容器中执行命令。
- `-i`：保持标准输入打开。
- `-t`：分配一个终端。
- `mysql`：容器名称。
- `bash`：在容器里执行的命令。

如果镜像里没有 `bash`，可以尝试换成 `sh`。

退出容器 shell：

```bash
exit
```

注意：`docker exec` 进入的是正在运行的容器。如果容器已经停止，需要先执行 `docker start`。


## 3. 数据卷

### 3.1 创建数据卷

```bash
docker volume create mysql-data
```

数据卷名不能重复。

### 3.2 查看所有数据卷

```bash
docker volume ls
```

### 3.3 查看数据卷详情

```bash
docker volume inspect mysql-data
```

### 3.4 删除数据卷

```bash
docker volume rm mysql-data
```

### 3.5 删除未使用的数据卷

```bash
docker volume prune
```

### 3.6 使用数据卷

```bash
docker run -d \
  --name mysql-demo \
  -v mysql-data:/var/lib/mysql \
  mysql:8.4
```

参数说明：

- `-v mysql-data:/var/lib/mysql`：将容器内目录挂载到数据卷，前者是 Docker 数据卷，后者是容器内目录。数据卷不存在时会自动创建。

## 4 目录挂载

目录挂载示例：

```bash
mkdir -p ~/docker-learning/nginx-html
echo "hello docker" > ~/docker-learning/nginx-html/index.html

docker run --name web-demo \
  -p 8080:80 \
  -v ~/docker-learning/nginx-html:/usr/share/nginx/html:ro \
  -d nginx:latest
```

格式：

```text
-v 宿主机路径:容器路径:选项
```

其中 `:ro` 表示只读挂载，容器不能修改宿主机目录内容。

学习时可以优先使用目录挂载，因为宿主机上的文件位置更直观。数据库等服务也可以使用 Docker 数据卷，减少误删宿主机目录的风险。

### 4.1 使用目录挂载替换 Nginx 默认页面

如果已经存在 `web-demo` 容器，先删除旧容器：

```bash
docker stop web-demo
docker rm web-demo
```

先在宿主机创建一个目录和页面文件：

```bash
mkdir -p ~/docker-learning/nginx-html
echo "hello docker" > ~/docker-learning/nginx-html/index.html
```

启动 Nginx，并把宿主机目录挂载到容器内：

```bash
docker run --name web-demo \
  -p 8080:80 \
  -v ~/docker-learning/nginx-html:/usr/share/nginx/html:ro \
  -d nginx:latest
```

参数说明：

- `-v ~/docker-learning/nginx-html:/usr/share/nginx/html:ro`：目录挂载。
- `~/docker-learning/nginx-html`：宿主机目录。
- `/usr/share/nginx/html`：容器内 Nginx 默认网页目录。
- `:ro`：read only，只读挂载，容器不能修改宿主机目录内容。

访问：

```text
http://127.0.0.1:8080
```

页面应显示：

```text
hello docker
```

## 5. 常见清理命令

### 5.1 删除已停止的容器

```bash
docker container prune
```

这个命令会删除所有已停止的容器。执行前先确认没有需要保留的容器：

```bash
docker ps -a
```

### 5.2 删除未被使用的镜像

```bash
docker image prune
```

这个命令会删除悬空镜像。悬空镜像通常是没有标签、没有容器使用的中间镜像。

### 5.3 查看 Docker 占用空间

```bash
docker system df
```

可以看到镜像、容器、数据卷和构建缓存占用的空间。

### 5.4 清理未使用资源

```bash
docker system prune
```

这个命令会清理未使用的容器、网络、悬空镜像和构建缓存。执行前确认没有需要保留的资源。

如果加上 `-a`，会删除所有未被容器使用的镜像，影响更大：

```bash
docker system prune -a
```
